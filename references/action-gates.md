# Action Gates

When a rep brings an action, or a list of them, run each through these four gates before it counts. This is the filter that separates a real action from action theater. An action that clears all four is leverage. One that fails gate 1 or 2 is usually busywork.

Where an action fails, do not supply the fix. Ask the question that makes the rep find it.

## The four gates

1. **Buyer or seller?**
   Does this advance a decision the buyer makes, or is it a seller task?
   - Seller task: "send the proposal," "follow up," "schedule a demo."
   - Buyer decision: "get the CFO the one number that unlocks her approval."
   - A seller task gets downgraded. Ask: "what buyer decision does this move?"

2. **Tied to the biggest risk?**
   Does this attack the single darkest gap on the deal, or work a part that is already healthy?
   - If the deal's dark gap is no economic buyer, building a nicer deck is busywork.
   - Ask: "of everything that could kill this deal, does this touch the most dangerous one?"

3. **What assumption is it betting on, and is that assumption evidenced?**
   Every action rests on a belief.
   - "Send the proposal" assumes they are ready to evaluate price. Are they? What is the evidence beyond the rep's gut?
   - Ask: "what do we believe is true for this to work, and what proof do we have?"

4. **Is it the highest-leverage move available?**
   Not the next obvious step. The one that moves win probability most.
   - If the rep cannot name a more leveraged option they considered and rejected, they probably defaulted to the obvious.
   - Ask: "what is the one move that changes the odds most, and is this it?"

## The output

- **Keep:** actions that clear all four.
- **Downgrade:** name the gate it failed and the probe that moves it forward.
- **Plays:** when a kept action is novel and worked, add it to the patch action rolodex so it can be installed into other deals.
